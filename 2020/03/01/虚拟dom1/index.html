# 虚拟DOM

通过变化侦测可以识别到数据的修改，之后通知外界进行视图的更改。使用虚拟DOM本质上就是通过运算较快的JS代替DOM操作，提升运行的速度。 Vue中通过编译将模版转换成渲染函数（rander），执行渲染函数生成虚拟节点树，再通过虚拟节点树渲染视图。

在vnode变化后，通过patch进行比较，以最小的代价修改节点。vnode是对各种节点的抽象类，即节点描述对象。vnode的类型有以下几种：

  1. 注释节点
  2. 文本节点
  3. 克隆节点
  4. 组件节点
  5. 元素节点
  6. 函数式节点

# patch

patch对现有DOM的操作主要为以下3种：

  1. 创建新增节点
    • 首次渲染时
    • vnode与oldVnode完全不是一个节点时
  2. 删除已经废弃的节点
    • vnode与oldVnode完全不是一个节点时
  3. 修改需要更新的节点
    • 新旧vnode是同一个时

# 更新节点的过程

  1. 判断节点的类型
  2. 如果节点相同或者为静态节点，不做处理
  3. 如果节点为文本节点，直接修改内容
  4. 如果新旧节点都包含子元素，更新子元素
  5. 如果新节点包含，旧节点不包含，清空旧节点数据，渲染新节点数据
  6. 如果旧节点包含，新节点不包含，清空节点

其中更新子元素的过程比较复杂，需要单独拿出来说。对于子节点的更新策略也包含新增、更新和删除操作。

  • 新增：迭代比较新旧数组，如果节点在旧数组中不存在，就添加新节点到未处理节点的前面。
  • 更新：如果发现节点的位置发生了变化，就将旧数组中的节点移到未处理数组的前面。
  • 删除：比较完所有的节点之后，舍弃到之前的数组节点。

# 优化策略

通过分析DOM操作的习惯，移位操作较少，Vue中使用了几种方法对迭代进行优化。在介绍优化策略之前需要先介绍几个指针。

  • newStartIndex： 新节点列表中未处理的节点的第一个的位置指针
  • newEndIndex： 新节点列表中未处理的节点的最后一个的位置指针
  • oldStartIndex： 旧节点列表中未处理的节点的第一个的位置指针
  • oldEndIndex： 旧节点列表中未处理的节点的最后一个的位置指针

相对应的优化策略也有以下几种：

  1. 首先比较newStartIndex和oldStartIndex，相同就同时向后移动一个指针
  2. 首先比较newEndIndex和oldEndIndex，相同就同时向前移动一个指针
  3. 首先比较newStartIndex和oldEndIndex，把oldEndIndex移动到对应位置，并做指针的移动
  4. 首先比较newEndIndex和oldStartIndex，移动旧节点到对应位置，做指针的移动

这样的优化策略是移动两边向中间移动的一个流程，减少了剩余节点的数量，降低了迭代的运算量。 之后不断根据之前迭代的思想不断向中间移动指针，知道新旧两个指针的其中一个迭代完。如果旧指针先结束，表示还有剩余的新增节点，统一进行新增的操作。 如果新指针先结束，旧指针剩下的项都是需要移除的节点。大大降低了运算量。
