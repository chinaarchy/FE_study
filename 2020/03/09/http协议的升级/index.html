# HTTP的优化

# 安全性

HTTP是明文发送，无法保证数据的私密性。同时也无法判断通信双方是不是真正通信的双方，以及通信过程中，信息有没有被改写。 以此提出了HTTPS协议，对安全性进行升级。

HTTP协议下直接和TCP协议进行通信，HTTPS在两者之间添加了SSL层。

# 加密方式

首先需要了解的是加密方式，通过加密的方式可以有效的保护数据的私密性。加密方式分为对称加密与非对称加密。

  • 对称加密方式：加密解密使用相同的密钥。
  • 非对称加密方式： 分为公钥和私钥，使用公钥进行加密，私钥进行解密

有了加密的工具后，就是对HTTP进行改造了。

首先，非对称加密的速度较慢（计算速度，网络传输的大小），为了保证更好的性能，采用两种加密方式结合的方式。

然后，初步的设想是两者通信通过非对称加密的方法确定通信双方的准确性互相认证，之后的内容使用对称加密的方法进行传递，保证数据的可靠性。

但是以上有一个问题，怎么确保最先通信时的非对称加密的公钥传送的可靠性呢。

# CA

为了解决公钥的准确性设立了证书认证机构也就是CA，通过他对每一个需要证书的公司进行公钥的加密，同时写入证书信息，在用户使用时依靠CA的可靠性完成所有的安全性通信。

最后的问题是CA的公钥怎么保证呢，之前总是觉得奇怪，最后发现是浏览器会自带常用的CA的数据，保证了可靠性。

# 降低通信数据与双向通信

# WebSocket

WebSocket可以完成双向通信的工作，两端握手的过程在头部设置Upgrade: websocket,在响应的握手过程中，响应码为101，即完成了握手，之后通过WebSocket进行通信。
