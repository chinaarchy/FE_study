# 事件相关实例方法

都在eventsMixin中进行挂载

  • $on

监听实例上的自定义事件，事件可以由$emit触发，实现方式如下。如果传入的是一个事件数组的话就循环递归调用，之后将事件注册到vm._events。

Vue.prototype.$on = function (event, fn) {
    const vm = this
    if (Array.isArray(event)) {
        for (let i = 0; i < event.length; i++) {
            this.$on(event[i], fn)
        }
    } else {
        (vm._events[event] || (vm._events[event] = [])).push(fn)
    }
    return vm
}

  • $off

其作用是移除自定义事件，设计上只需要对绑定上的事件组进行迭代找到对应的事件进行移除，其中有一个细节，在数组中进行循环的过程中，采用从后向前的方式。 保证在下次循环中不会出现问题，跳过一个元素。

  • $once

其作用也是注册一个自定义事件，但是只触发一次。其设计思路就是结合上面两个函数。

Vue.prototype.$once = function (event, fn) {
    const vm = this

    function on() {
        vm.$off(vm, on)
        fn.apply(vm, arguments)
    }

    on.fn = fn
    vm.$on(event, on)
    return vm
}

  • $emit

触发事件，代码如下：

Vue.prototype.$emit = function (event) {
    const vm = this

    let cbs = vm._events[event]

    if (cbs) {
        const args = toArray(arguments, 1)
        for (let i = 0; i < cbs.length; i++) {
            try {
                cbs[i].apply(vm, args)
            } catch (e) {
                warn(e)
            }
        }
    }
    return vm
}

# 生命周期相关实例方法

分别有4个相关函数，其中$forceUpdate和$destory在lifeCycleMixin中挂载，$nextTick在renderMixin中挂载，$mount在跨平台部分进行挂载。

  • $forceUpdate 其通过调用Watcher的更新方法，更新整个组件
  • $destory删除组件，不仅需要删除父组件内的连接，同时还需要删除连接的各类关系。并且在执行之前和之后调用生命周期函数。
  • $nextTick在渲染任务完成后执行内部数据，一般使用微任务进行注册。
  • $mount挂载到对应节点

# 全局API

  • Vue.extend创建Vue子类
  • Vue.dieective创建自定义指令，将自己定义的指令绑定到options上
  • Vue.filter自定义过滤器，在括号插值和v-bind中进行
  • Vue.component创建自定义组件同样绑定到options上
  • Vue.use安装Vue插件，首先需要判断是否为对象并且包含install函数，或者传入函数将其视为install函数，同时为了保证不被重复注册，建立对应的hash。
  • Vue.mixin注册全局混入，将用户传入的数据与Vue进行合并
