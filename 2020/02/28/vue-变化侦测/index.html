# Vue的变化侦测

MVVM框架通过vm进行变化的侦测，降低了MVC结构之间的耦合，并且大幅度的提升了开发效率。

关于Vue的变化侦测分为两个部分：对象的变化侦测和数组的变化侦测。本文介绍对象的变化侦测的实现。

# Object的变化侦测

# 如何追踪变化

由于ES6的支持不够理想，故Vue选择了defineProperty，没有使用Proxy。两者构造响应式系统的原理相同，Vue3也有使用Proxy进行重写

function defineReactive (data, key, val) {
    Object.defineProperty(data, key, {
        enumerable: true,
        configurable: true,
        get: function () {
            return val
        },
        set(newValue) {
            if(val === newValue){
                return
            }
            val = newValue
        }
    })
}

# 如何收集依赖

下面解决如何收集依赖，思路上也很简单，在get中收集依赖，在set中触发依赖。


 




 






 
 
 





function defineReactive (data, key, val) {
    let dep = []
    Object.defineProperty(data, key, {
        enumerable: true,
        configurable: true,
        get: function () {
            dep.push(window.target)
            return val
        },
        set(newValue) {
            if(val === newValue){
                return
            }
            for(let i = 0; i < dep.length; i++){
                dep[i](newValue, val)
            }
            val = newValue
        }
    })
}

为了降低耦合，将收集依赖的功能封装成一个类

class dep {
    constructor(){
        this.subs = []
    }
    addSub(sub){
        this.subs.push(sub)
    }
    removeSub(sub){
        remove(this.subs, sub)
    }
    depend(){
        if(window.target){
            this.addSub(window.target)
        }
    }
    notify(){
        const subs = this.subs.slice()
        for(let i = 0; i < subs.length; i++){
            subs[i].update()
        }
    }
}
function remove(arr, item) {
    if(arr.length){
        const index = arr.indexOf(item)
        if(index > -1){
            return arr.splice(index, 1)
        }
    }
}

# 依赖是谁

收集的依赖就是当数据发生变化时通知的对象。通知对象的类型较多，可能是模版，也可能是用户写的一个watch，需要抽象出一个统一的类。

class Watcher {
    constructor(vm, expOrFn, cb) {
        this.vm = vm
        this.getter = parsePath(expOrFn)
        this.cb = cb
        this.value = this.get()
    }

    get() {
        window.target = this
        let value = this.getter.call(this.vm, this.vm)
        window.target = undefined
        return value
    }
    
    update() {
        const oldValue = this.value
        this.value = this.get()
        this.cb.call(this.vm, this.value, oldValue)
    }
}

已经实现了依赖的对象以及收集依赖的方法，下一步就要实现一类完成监测一个数据内所有的属性

class Observer {
    constructor(value){
        this.value = value
        if(!Array.isArray(value)){
            this.walk(value)
        }
    }
    walk(obj){
        const keys = Object.keys(obj)
        for(let i = 0; i< keys.length; i++){
            defineReactive(obj, keys[i], obj[keys[i]])
        }
    }
}

以上实现了对对象的变化监测，但是defineProperty没有元编程的能力，无法监测到例如对象中添加或删除属性等。Vue中使用$set和$delete解决问题。

# 总结

为了实现对象的变化侦测，当数据发生变化可以侦测并发出通知，根据一下思路进行设计。

  1. 对象可以使用defineProperty进行侦测变化，在读取数据时触发getter，在修改数据时触发setter
  2. 为了存放收集的依赖，构造了dep类，收集、删除、通知依赖
  3. 存放的依赖类型较多，故抽象成一个依赖类
  4. 为了将数据中的全部属性都变成响应式的构造Observer类。
